Designing a Program

The process of designing a program can be summarised in following two steps

  1. Understand the task the program is to perform
  2. Determine the steps to be taken to perform the task

Description

Key Concepts:

  1. Input, Process, Output:
    • Programs receive input, process it (e.g., calculations), and provide output.
    • Input is stored in variables for further manipulation.
  2. Variables
    • Variables are storage locations in memory with specific names
    • Variables are declared with a name and a data type
    • Examples of valid variable names: grossPay, pay_rate, length
  3. Variable Declarations and Data Types
    Integer
    Real
    String
    Example of variable declaration: Declare Integer length.
  4. Prompting the User
    • Programs prompt users to input data using messages like "Enter your age."
    • Example in pseudocode: Input age.
  5. Assignments and Calculations:
    • Assignment statements set a variable's value.
    • Common operators:
      • +:Addition
      • -:Subtraction
      • *:Multiplication
      • /:Division
      • %:Modulus (remainder after division)
      • ^:Exponent
  6. Order of Operations
    1. Parentheses
    2. Exponentiation
    3. Multiplication, Division, Modulus (left to right)
    4. Addition, Subtraction (left to right)
    Example: 12 + 6 / 3 evaluates to 14.
  7. Data Type Compatibility:
    • Integer variables cannot hold real numbers.
    • Real variables can hold integers.
  8. Integer Division:
    • Integer division results in an integer by discarding the fractional part (truncation).
    • Example: 3 / 2 stores 1 in an integer variable
  9. Constants
    • A constant's value cannot change during program execution.
    • Example: Constant Real INTEREST_RATE = 0.069.

Output, Input and Variables

  • Data produces and displayed by the program is called output
  • Data received by the program is called Input.
  • When a program receives data, it stores it in a variable. A variable is a named storage location in memory.
  • Input is data that is typed on the keyboard. Once input is received, some process, such as a mathematical calculation, is usually performed on it. The results of the process are then sent out of the program as output.

Variables

  • A program needs to store data in the computer’s memory
  • When you browse a Website and add the items to the shopping cart, data about those items is stored in memory.
  • When you click the checkout button, a program running on the website’s computer calculates the total of all the items in your shopping cart, applicable sales taxes, shopping cose, and the total of all these charges.
  • A variable is a storage location in memory that is represented by a name.

Naming schemes

  • Variable names must be one words. They cannot contain spaces.
  • In most languages, punctuation characters cannot be used in variable names.
  • Use only alphabetic letters and numbers in variable names. The first character of a variable cannot be a number.
  • Valid Identifiers
    • grosspay payrate
    • gross_pay pay_rate
    • grossPay payRate

Example schemes

  • flatcase
  • camelCase
  • kebab-case
  • Snake_case
  • PascalCase

Declarations and Data types

  • Variable declaration is a statement that specifies two things about a variable
    • The variable’s name and data type
  • A variable’s data type is the type of data that the variable will hold.
  • The data types that you are allowed to use depend on the programming language. For example, the Java language provides four data types for integer numbers, two data types for real numbers, one data type for strings and others.
  • A variable of Integer can hold whole numbers, such as 42,0,-99
  • A variable of Real data type can hold either whole numbers or numbers with fractional parts, such as 3.5, and 3.0.
  • A variable of the String data type can hold any string of character, such as name, address, password, and so on.
