What is a file, and a file system?

A file is the term used for any collective unit of stored data and must have a unique identifier on any storage device. Every time you format a hard-disk or flash-drive, you're writing the filesystem to one or more partition (section of the storage drive). These filesystems determine how data or information from the user is stored by allowing you to save, write, and read those files in an organized structure (data structure).

What is a hierarchical file system? How is it organized?

A hierarchical file system collects data into a file unit, then sorts those files in a pyramid—like a family tree. There are several types of files in Linux management: Regular Files, Directories, and Special files, Pipes, Sockets, and Symbolic Links:

  • Regular files just contain—not limited to—text binary, videos and images
  • Directories are a file-type that store file types and their information.
  • Special Files represent physical devices used for IO operations
  • A Pipe is a temporary file that holds output data from one command to be used as input for a second command separated by a pipe symbol (|)
  • A Symbolic Link is a reference to another file. (aka Soft link) It has text locating the file path to the file it references.

Instead of parents, grandparents, and children, the Linux file system organizes directories into a tree structure with root (/) as the parent of all directories. All files require a unique identifier called a path name, which specifies the location of a file in the system through the names of the file itself, and the directories over it. Under the root, there is a specific layout of directories that organizes data within a scope of visibility and purpose.

/dev
Special files store the device data as blocks (block special data) or a stream (character special data) under a subdirectory of root called dev.
/boot
Contains the files to boot the system
/etc
Machine-local system configuration files
/home
User home directories

/bin vs /sbin

They are both essential to the filesystem, but note that:

/bin
stores the command binaries for running the system
/sbin
stores the utilities of system administration
